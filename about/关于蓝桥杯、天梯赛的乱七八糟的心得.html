    # 关于蓝桥杯、天梯赛的乱七八糟的心得

    # 蓝桥杯

    先介绍一下自身情况，我参加的是C/C++B组，所在赛区是山西赛区，参加的是第十三届（2022年）。

    今年的题目只有两道填空题，剩下的全部是程序大题，往年都会有五道填空，填空与大题一半一半，（我自认为是为了防作弊）。并且我感觉今年的题很难（可能因为我是个小辣鸡）。

    今年的题目考的dfs变少了，更多考的是贪心，动态规划，还喜欢考数学知识（我一整个寒假都在洛谷上刷dfs，结果到了考场上还是不太会）。并且蓝桥杯的dfs是解决一个实际问题的dfs，而不是搜索地图的那种dfs，一般是不用建立坐标的那种。蓝桥杯和天梯赛相比，蓝桥杯考的东西感觉很杂，并且感觉自己在复习的过程中是有点没有目的，抓不准大纲。

    而且蓝桥杯的特点是提交之后无法看到自己正不正确，可能自己感觉是对的，跑题目给定案例也是对的，但是交上去其实是错的，（我自己感觉，题目给的案例是有魔法的，就算我的代码错的再离谱，跑题目中的案例永远是对的😅）。

    其实有些题目如果想不出来正确做法，我一般都会用暴力，祈求过一点测试样例，但是今年的题的数据范围都特别大，估计拿不了多少分。

    # 天梯赛

    天梯赛和蓝桥杯的感觉完全不一样，它分为L1,L2,L3三个难度，L1主要就是模拟，不会牵扯到算法，L2主要考数据结构，但这里的数据结构不需要自己实现，用C++的STL就好了，主要就是用队列，集合，map，栈。还会考图论，并查集和二叉树，图论里又分为迪杰斯特拉算法，spfa算法，弗洛伊德算法等。L3就是真正的考算法了，（我一般是不会去看L3，因为我连L2都弄不完）。

    我觉得天梯赛比较好的一点就是，它的考察范围很清楚，而且题库也很清楚，直接刷天梯赛合集和PTA乙级就好，如果好好准备天梯赛几个月，是有可能拿天梯赛个人省级奖的。

    在我得知要参加天梯赛的时候，已经没多少时间了，而且天梯赛和蓝桥杯的考点相差很大，蓝桥杯不会考图论那些东西，所以现学已经来不及了，只能专心把L1复习完，L2把涉及到set，map，queue的那些做一做。

    # 总结

    如果给我一次重新选择的机会，我一定会将重心放在天梯赛上，我感觉，好好准备天梯赛是有可能拿省奖的，而且天梯赛的排名是实时的，每个题是可以看到自己到底对不对的，我个人更喜欢可以看到自己那个题是否正确。而蓝桥杯完全不知道，它的考察范围也很广。

    蓝桥杯的监考措施好像不够完善，今年抓到了好多作弊的，天梯赛的监考很严格，但是严格监考的代价就是要很早的进入考场，基本上从上午十一点半进考场，下午一点半开考，四点半结束，中午不休息，到了最后，整个人已经麻木了，感觉自己的精力已经耗尽，一点都不想敲代码了。
